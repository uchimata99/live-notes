# מנגנון הערות חי — `live-notes`

ווידג'ט קטן ועצמאי שמאפשר להוסיף **הערות חיות** על כל עמוד אינטרנט, לסמן עם עט,
ולשלוח את הכול כ‑**GitHub Issue** — שאותו סוכן Claude Code מעבד לתיקונים.
בלי שרת, בלי טוקנים, בלי תלות. קובץ JS אחד.

מתוכנן לשתילה בעוד פרויקטים (למשל דשבורדים): שולחים הערות תוך כדי עבודה,
הן מצטברות כ‑Issues, וכשרוצים — Claude מריץ את כולן.

---

## התקנה בפרויקט (שני שורות)

```html
<script>window.LIVE_NOTES_CONFIG = { repo: 'OWNER/REPO' };</script>
<script src="https://uchimata99.github.io/live-notes/live-notes.js"></script>
```

או, לפרויקט "קובץ יחיד" בלי תלות חיצונית — פשוט מעתיקים את תוכן `live-notes.js`
לתוך תגית `<script>` בעמוד, ומגדירים `window.LIVE_NOTES_CONFIG` לפניו.

### הפעלה (כדי שמשתמשי קצה לא יראו אותו)
הווידג'ט **רדום כברירת מחדל**. כדי להפעיל אותו במכשיר שלך, היכנס פעם אחת לכתובת עם
`?notes=1` (הדגל נשמר ב‑localStorage). לכיבוי: `?notes=0`. להצגה תמידית: `always:true` בהגדרות.

### הגדרות (`window.LIVE_NOTES_CONFIG`)
| מפתח | ברירת מחדל | תיאור |
|---|---|---|
| `repo` | — (חובה) | `owner/name` — המאגר שאליו נשלחים ה‑Issues |
| `label` | `'note'` | תווית ה‑Issue (לסינון בעיבוד) |
| `title` | `'הערות'` | כותרת הפאנל |
| `accent` | `'#c08bff'` | צבע מבדיל |
| `dir` | `'rtl'` | כיוון |
| `always` | `false` | `true` = להציג תמיד |
| `locationFn` | כותרת/hash | פונקציה שמחזירה את "המיקום" שמצורף לכל הערה (התאם לפרויקט) |

## איך Claude מעבד את ההערות (צד הסוכן)

ההערות מגיעות כ‑Issues עם התווית שהוגדרה. בסשן **Claude Code** עם המאגר, כשהמשתמש
אומר **"תריץ את ההערות"**:

```bash
gh issue list  --repo OWNER/REPO --label note --state open
gh issue view  <מספר> --repo OWNER/REPO --json title,body
# ... החל כל הערה בגוף, אמת, commit, push ...
gh issue close <מספר> --repo OWNER/REPO --comment "תוקן ✓"
```

מבנה גוף ה‑Issue:

```
הערות — OWNER/REPO
========================

[מיקום ההערה]
• מה לתקן

[מיקום נוסף]
• ...
```

**כללים לסוכן:** קרא קודם את ה‑`CLAUDE.md`/הוראות הפרויקט; שמור על העקרונות שלו;
אמת לפני כל commit; אל תיגע בנתוני זמן‑ריצה (מפתחות `livenotes_*` / `ln_*` ב‑localStorage —
אלה של כלי ההערות).

### אוטומציה מלאה (רשות)
אפשר להגדיר שגרה (cron / routine) שמריצה תקופתית `gh issue list --label note --state open`
ומעבדת את התור לבד. היתרון: אפס טריגר ידני. החיסרון: שינויים עולים בלי סקירה מוקדמת.
מומלץ להשאיר גם את הטריגר הידני "תריץ את ההערות".
